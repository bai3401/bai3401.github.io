Hexo

环境

  • Nodejs
  • Git

安装

-g 表示全局安装

初始化

安装主题 Butterfly

使用npm安装后主题文件保存在node_modules/hexo-theme-butterfly文件夹内

使用git命令安装在themes/butterfly文件夹下

应用主题

在_config.yml文件夹下修改

Butterfly主题依赖 pug 以及 stylus 的渲染器，所以需要用npm安装

将Butterfly主题文件夹下的_config.yml更名为_config.butterfly.yml并复制到网站根目录下

Hexo会自动合并主题中的config.yml和 _config.butterfly.yml里的配置，如果存在同名配置，会使用config.butterfly.yml的配置，其优先度较高。并且这样使用升级主题的时候，可以避免原来的配置文件被覆盖掉而丢失。

相关配置

网站配置（_config.yml）

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述
keywords 网站的关键词。支持多个关键词。
author 您的名字
language 网站使用的语言。对于简体中文用户来说，使用不同的主题可能需要设置成不同的值，请参考你的主题的文档自行设置，常见的有 zh-Hans和 zh-CN。Butterfly是zh-CN
timezone 网站时区。Hexo 默认使用您电脑的时区。请参考 时区列表 进行设置，如 America/New_York, Japan, 和 UTC 。
一般的，对于中国大陆地区可以使用 Asia/Shanghai。
url 网址, must starts with http:// or https://
permalink 文章的 永久链接 格式
theme 当前主题名称。值为false时禁用主题
deploy 部署部分的设置

域名和加速配置（Vercel）

什么是Vercel

vercel 一个由next.js团队打造的Serverless平台。简单来说他和gitpage很像，你可以通过类似hexo这样静态模板技术来部署一套属于自己的个人博客。

为什么选择Vercel

GithubPages容易打不开，而且慢，Vercel更快，延迟更低。

Vercel配置

  1. 打开vercel官网：Develop. Preview. Ship. For the best frontend teams – Vercel，使用Github账号登陆

  2. 授权完成后，点击Create a New Object

  3. 找到博客项目，点击import

  4. 等待部署完成

  5. 进入DashBoard
